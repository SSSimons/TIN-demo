# TIN-Demo

**TIN-Demo** — настольное учебно-демонстрационное ГИС-приложение на Python и PyQt6 для загрузки векторных данных, построения триангуляционной нерегулярной сети (**TIN**), генерации горизонталей и визуальной оценки высоты поверхности.

Программа умеет открывать контурные линии из нескольких распространённых геоформатов. Если исходные данные содержат высоты, приложение строит триангуляцию Делоне и автоматически создаёт изолинии. Если высот нет, геометрия отображается как обычный векторный слой.

## Возможности

- импорт DXF, ESRI Shapefile, GeoJSON и NDJSON;
- чтение GeoJSON из архивов `.gz` и `.zip`;
- поддержка линейной и полигональной геометрии;
- получение высоты из координаты Z или атрибутов объекта;
- построение TIN по исходным высотным линиям;
- автоматическая генерация горизонталей методом Marching Triangles;
- интерполяция высоты внутри треугольника по барицентрическим координатам;
- отображение высоты под курсором;
- масштабирование карты колесом мыши относительно положения курсора;
- ручное рисование ломаных линий;
- отмена и повтор добавления нарисованных линий;
- сохранение и открытие проектов в формате `.mapproj`;
- фоновая загрузка данных без блокировки интерфейса;
- встроенная HTML-справка.

## Поддерживаемые форматы

| Формат | Расширения | Поддерживаемые данные |
|---|---|---|
| DXF | `.dxf` | `LWPOLYLINE`, высота из свойства `elevation` |
| ESRI Shapefile | `.shp` | PolyLine, PolyLineZ, Polygon и PolygonZ |
| GeoJSON | `.geojson`, `.json` | LineString, MultiLineString, Polygon, MultiPolygon |
| NDJSON / потоковый JSON | `.json` | Последовательность GeoJSON-объектов |
| GZip | `.gz` | Сжатый GeoJSON или JSON |
| ZIP | `.zip` | Первый найденный внутри `.json` или `.geojson` |
| Проект TIN-Demo | `.mapproj` | Исходные линии и данные TIN в JSON-структуре |

### Определение высоты

Для GeoJSON и Shapefile программа сначала использует третью координату точки, если она присутствует. В остальных случаях высота ищется в одном из полей:

```text
elev, ELEV_FT, ELEV_M, ELEVATION, Z, Contour
```

Если высота не найдена, она считается равной `0`. Набор, в котором все точки имеют нулевую высоту, отображается без построения TIN и горизонталей.

## Требования

- Python **3.10 или новее**;
- операционная система с поддержкой PyQt6 и OpenGL;
- зависимости из `requirements.txt`.

Основные библиотеки:

- `PyQt6` — графический интерфейс;
- `NumPy` и `SciPy` — обработка координат и триангуляция Делоне;
- `Shapely` — работа с геометрией;
- `Fiona` и `pyshp` — чтение Shapefile;
- `ezdxf` — чтение DXF;
- `ijson` — потоковый разбор JSON;
- `triangle` и `numba` — резервные варианты построения TIN;
- `PyOpenGL` — OpenGL-зависимость виджета карты.

> Некоторые геопространственные библиотеки содержат нативные компоненты. На отдельных системах для установки Fiona, Rasterio или GDAL-зависимостей могут потребоваться готовые бинарные пакеты либо системные библиотеки.

## Использование

### Загрузка данных

1. Выберите **Файл → Открыть…** или нажмите `Ctrl+O`.
2. Откройте векторный файл либо проект `.mapproj`.
3. Программа загрузит данные в отдельном потоке.
4. При наличии высот будет построен TIN и сгенерированы горизонтали.
5. При отсутствии высот будут показаны только исходные линии.

После импорта строка состояния показывает количество линий, точек или треугольников.

### Навигация

Вращайте колесо мыши для изменения масштаба. Масштабирование выполняется относительно текущего положения курсора.

При первой загрузке объектов приложение автоматически подбирает преобразование координат так, чтобы данные поместились в окне.

### Просмотр высоты

После построения TIN наведите курсор на поверхность. В нижней части карты появится интерполированная высота в метрах.

Значение рассчитывается по вершинам треугольника с помощью барицентрической интерполяции.

### Ручное рисование

1. Включите режим через **Правка → Рисовать изолинию** или `Ctrl+D`.
2. Добавляйте вершины левой кнопкой мыши.
3. Завершите линию двойным щелчком левой кнопки мыши.
4. Нарисованная линия будет добавлена в список исходных линий и в стек отмены.
5. Повторно нажмите `Ctrl+D`, чтобы выключить режим рисования.

### Сохранение проекта

Выберите **Файл → Сохранить проект…** или нажмите `Ctrl+S`. Проект сохраняется в JSON-подобном формате с расширением `.mapproj`.

## Как работает программа

Общий конвейер обработки данных:

```text
Векторный файл
      ↓
Parser / GeoJSON streaming reader
      ↓
Polyline и Point2D
      ↓
Подготовка и упрощение геометрии
      ↓
Триангуляция Делоне
      ↓
TIN: список Triangle
      ↓
Marching Triangles
      ↓
Горизонтали и отображение на MapCanvas
```

### Построение TIN

Класс `Triangulator` последовательно пытается использовать три реализации:

1. `scipy.spatial.Delaunay` на базе Qhull;
2. библиотеку `triangle`;
3. собственную реализацию алгоритма Bowyer–Watson, ускоряемую `numba`, если она доступна.

Перед триангуляцией линии объединяются, упрощаются алгоритмом Дугласа — Пекера и преобразуются в набор точек.

### Построение горизонталей

`IsoContourGenerator` перебирает уровни высоты с заданным шагом, пересекает каждый треугольник плоскостью `z = h`, получает отрезки и соединяет их в полилинии.

В графическом интерфейсе шаг горизонталей сейчас задан в коде и равен **10 единицам высоты**.

### Интерполяция высоты

Модуль `model/baryinterpol.py` вычисляет барицентрические координаты точки внутри треугольника и интерполирует значение Z по высотам трёх вершин.

## Программное использование

Основные компоненты можно использовать без главного окна:

```python
from datainput.parser import read_vector
from model.triangulator import Triangulator
from model.isocontour import IsoContourGenerator

lines = read_vector("contours.geojson", as_polyline=True)

tin = Triangulator(lines)
tin.build()

isolines = IsoContourGenerator(tin.triangles).generate(step=10)

print(f"Точек: {len(tin.points)}")
print(f"Треугольников: {len(tin.triangles)}")
print(f"Горизонталей: {len(isolines)}")
```

Экспорт линий в GeoJSON доступен через функцию `save_as_geojson`:

```python
from datainput.parser import save_as_geojson

save_as_geojson(isolines, "isolines.geojson")
```

После установки зависимостей тесты запускаются командой:


Проверка покрытия:

```bash
coverage run -m pytest

```

### Состояние тестов в текущем архиве

Тестовый набор был написан для более ранней версии внутреннего API и требует синхронизации с текущими классами:

- тесты импортируют `Point`, тогда как актуальная модель называется `Point2D`;
- часть тестов передаёт в `Triangulator` обычные списки точек вместо `Polyline`;
- тест изолиний использует старые имена аргументов и атрибутов.

Синтаксис всех Python-файлов успешно проверяется через `compileall`, но перед подключением CI тесты необходимо обновить.

## Ограничения текущей версии

- шаг автоматически создаваемых горизонталей фиксирован в интерфейсе и равен `10`;
- панорамирование карты мышью пока не реализовано;
- DXF-импорт обрабатывает только `LWPOLYLINE`;
- ZIP-импорт выбирает первый найденный файл `.json` или `.geojson`;
- точки, полигоны без границы и другие типы геометрии не отображаются;
- рисуемым вручную линиям назначается высота `0`;
- экспорт в GeoJSON реализован как Python-функция, но не добавлен в меню;
- сохранение TIN через графический интерфейс требует доработки: текущий обработчик передаёт в `ProjectManager` список треугольников вместо экземпляра `Triangulator`;
- встроенная справка частично описывает команды завершения рисования клавишами `C` и `X`, однако в текущем коде завершение реализовано только двойным щелчком.
